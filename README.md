# Волшебная живая баня

Современный адаптивный сайт русской бани на дровах в Калуге: аренда дом-бани в Колюпаново, сибирский чан, купель, индивидуальное парение, мангальная зона и отдых для компании до 10 гостей.

Сайт: [banya-kalyga.ru](https://banya-kalyga.ru/)

Карта: [Яндекс Карты](https://yandex.ru/maps/org/volshebnaya_zhivaya_banya/110090709629/)

Связь: [+7 910 510-53-48](tel:+79105105348), [MAX](https://web.max.ru/-72553877540105)

## Что внутри

- Первый экран с продающим оффером, CTA-кнопками, рейтингом и быстрыми фактами.
- Адаптивная верстка для desktop, tablet и mobile.
- SEO-разметка: `title`, `description`, Open Graph, Twitter Card, canonical, geo meta, `robots.txt`, `sitemap.xml`.
- JSON-LD schema.org для локального бизнеса, адреса, рейтинга и каталога услуг.
- Галерея с модальным просмотром, миниатюрами, перелистыванием и свайпами.
- Плавающий аудиоплеер с раскрытием, мини-плеером, прогрессом, громкостью и списком треков.
- Ссылки на Яндекс Карты во всех адресных точках.
- PWA-манифест, favicon и мобильные иконки.
- GitHub Pages workflow для автоматической публикации из ветки `main`.

## Технологии

Проект сделан как статический сайт без сборщика:

- `HTML5`
- `CSS3`
- чистый `JavaScript`
- `lucide` для иконок через CDN
- GitHub Pages для публикации

## Структура

| Путь | Назначение |
| --- | --- |
| `index.html` | Контент страницы, SEO meta, schema.org, секции сайта |
| `styles.css` | Визуальная система, адаптив, анимации, компоненты |
| `script.js` | Меню, галерея, scroll-анимации, аудиоплеер |
| `assets/images/` | Фотографии и Open Graph изображение |
| `assets/audio/` | Треки для встроенного аудиоплеера |
| `assets/icons/` | Иконки сайта и PWA |
| `favicon.svg` | Векторная иконка сайта |
| `site.webmanifest` | Настройки PWA/иконок |
| `robots.txt` | Правила индексации |
| `sitemap.xml` | Карта сайта с изображениями |
| `.github/workflows/static.yml` | Автопубликация на GitHub Pages |
| `RELEASES.md` | История релизов и порядок выпуска версий |

## Локальный запуск

Сайт можно открыть как статический файл, но для проверки путей к ассетам лучше запускать локальный сервер:

```bash
python -m http.server 8080
```

После запуска открыть:

```text
http://localhost:8080/
```

На Windows также можно использовать:

```powershell
py -m http.server 8080
```

## Проверка перед публикацией

Минимальный чек-лист:

1. Проверить главную страницу на desktop и mobile.
2. Открыть галерею, перелистнуть фото стрелками и свайпом.
3. Включить аудио, проверить мини-плеер, прогресс, громкость и список треков.
4. Проверить CTA: телефон, MAX, Яндекс Карты.
5. Проверить отсутствие горизонтального скролла на ширине 320px.
6. Проверить синтаксис JS:

```bash
node --check script.js
```

## Публикация

Публикация настроена через GitHub Pages. Любой push в ветку `main` запускает workflow:

```text
.github/workflows/static.yml
```

Обычный порядок работы:

```bash
git add .
git commit -m "Краткое описание изменения"
git push origin main
```

## Релизы

История версий ведется в [RELEASES.md](RELEASES.md). Для крупного обновления:

```bash
git tag -a v1.0.0 -m "Релиз v1.0.0"
git push origin v1.0.0
```

Версии оформляются по смыслу изменений:

- `major` - большой редизайн, смена структуры, новая версия сайта.
- `minor` - новые секции, крупные функции, новые интеграции.
- `patch` - правки текста, верстки, SEO, небольшие улучшения.

## Безопасность

Доступы, пароли, токены, cookie и любые секреты нельзя хранить в репозитории. Для локальных доступов используются только файлы вида `.env.local`, которые исключены из git.
